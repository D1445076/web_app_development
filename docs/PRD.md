## 1. 專案概述

### 1.1 背景與動機

在日常生活與工作中，個人常需同時追蹤多項待辦事項，若缺乏有效的組織工具，容易造成遺忘或優先順序混亂。本系統旨在提供一個簡單、直覺的任務管理介面，讓使用者能快速新增、追蹤及完成日常待辦項目，不需複雜的帳號設定或學習成本。

### 1.2 目標用戶

- **主要用戶：** 需要管理日常工作的個人用戶（學生、上班族、自由工作者）
- **技術背景：** 不需具備技術能力，熟悉基本網頁操作即可
- **使用情境：** 每日規劃工作清單、追蹤進度、完成後標記任務

### 1.3 核心價值主張

> 「用最少的操作步驟，清楚掌握每一件待辦事項的狀態。」

- **簡單易用：** 介面直覺，新增任務只需輸入文字並送出
- **即時回饋：** 完成與刪除操作立即反映，無需重整頁面
- **靈活篩選：** 依完成狀態篩選任務，幫助使用者聚焦當前需處理的工作

## 2. 功能需求

### 2.1 主要功能清單與使用者故事

| # | 功能名稱 | 使用者故事 |
|---|----------|------------|
| F01 | 新增任務 | 作為**個人用戶**，我希望**在輸入框中輸入任務描述並按下新增**，以便**快速記錄待辦事項而不中斷目前的工作流程**。 |
| F02 | 標記任務完成 | 作為**個人用戶**，我希望**點擊任務旁的完成按鈕或核取方塊**，以便**標記該任務已完成並與未完成任務做出視覺區別**。 |
| F03 | 刪除任務 | 作為**個人用戶**，我希望**點擊任務旁的刪除按鈕**，以便**移除不再需要追蹤的任務，保持清單整潔**。 |
| F04 | 顯示所有任務清單 | 作為**個人用戶**，我希望**進入系統後立即看到所有任務的清單**，以便**快速掌握所有待辦與已完成事項的全貌**。 |
| F05 | 依完成狀態篩選 | 作為**個人用戶**，我希望**透過篩選按鈕切換「全部 / 待完成 / 已完成」**，以便**聚焦於特定狀態的任務，減少視覺干擾**。 |

### 2.2 功能說明細節

#### F01 — 新增任務
- 頁面頂部提供文字輸入框與「新增」按鈕
- 輸入框為空時禁止送出，並顯示提示訊息
- 成功新增後清空輸入框，新任務出現在清單最頂端
- 任務預設狀態為「未完成」

#### F02 — 標記任務完成
- 每筆任務旁提供核取方塊（checkbox）或完成按鈕
- 標記完成後，任務文字顯示刪除線樣式以示區別
- 再次點擊可取消完成（切換回未完成狀態）

#### F03 — 刪除任務
- 每筆任務旁提供刪除按鈕（垃圾桶圖示）
- 刪除前可選擇性加入確認提示（考量 UX 流暢性，預設直接刪除）
- 刪除後頁面立即更新

#### F04 — 顯示所有任務清單
- 進入系統主頁即顯示完整任務清單
- 清單依建立時間排序（最新在最上方）
- 每筆任務顯示：任務名稱、完成狀態、操作按鈕

#### F05 — 依完成狀態篩選
- 提供三個篩選選項：「全部」、「待完成」、「已完成」
- 篩選為前端即時切換，不重新載入頁面（或後端 Query 均可）
- 預設顯示「全部」

## 3. 非功能需求

### 3.1 技術限制

| 類別 | 規格 |
|------|------|
| 後端框架 | Python Flask |
| 模板引擎 | Jinja2 |
| 資料庫 | SQLite（輕量，無需額外安裝） |
| 前端 | HTML + Vanilla CSS + 少量 JavaScript |
| 部署環境 | 本機開發（`flask run`），後續可部署至 PythonAnywhere 等平台 |

### 3.2 效能考量

- 系統為單人使用的本機應用，不需考慮高並發
- 頁面首次載入時間應在 **1 秒以內**
- 任務操作（新增、完成、刪除）應在 **500ms 以內** 完成回應

### 3.3 安全考量

- 使用 SQLAlchemy ORM 或參數化查詢，防止 **SQL Injection**
- 使用 Jinja2 自動跳脫機制，防止 **XSS 攻擊**
- 本版本為單人本機使用，暫不實作身份驗證（Login）

### 3.4 使用者體驗

- 介面需手機與桌機皆可使用（RWD 響應式設計）
- 操作流程簡單，核心功能不超過 3 步驟完成
- 系統需有空狀態提示（例如：無任務時顯示「目前沒有任何任務，立即新增！」）
